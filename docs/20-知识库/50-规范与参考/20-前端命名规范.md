# 前端命名规范

> 来源：[[00-项目总览|项目 VitePress 文档]]
>
> 适用于 **Vue 3 + Vite + TypeScript + Element Plus** 项目（spectra-ui / spectra-app）。

## 总体原则

1. **文件名一律 `kebab-case`（短横线命名）**
2. **目录名一律 `kebab-case`**
3. **Vue 组件名使用 `PascalCase`**
4. **导出对象/类名 `PascalCase`，函数 `camelCase`**

## 推荐目录结构

```
src/
├── api/              # 接口定义层
├── assets/           # 静态资源
├── components/       # 全局公共组件
├── directive/        # 自定义指令
├── hooks/            # 组合式函数
├── plugin/
│   ├── element/      # Element Plus 全局配置
│   ├── request/      # HTTP 封装
│   ├── router/       # 路由配置
│   └── stores/       # Pinia 初始化
├── utils/            # 通用工具类
├── views/            # 页面组件
├── App.vue
└── main.ts
```

## 文件命名规则

| 类型 | 规范 | 示例 |
|---|---|---|
| 普通 TS/JS | `kebab-case.ts` | `message-helper.ts` |
| Vue 组件 | `PascalCase.vue` | `UserTable.vue` |
| composables | `use-xxx.ts` | `use-message.ts` |
| API 文件 | `xxx-api.ts` | `role-api.ts` |
| Store 文件 | `xxx-store.ts` | `user-store.ts` |
| 页面文件 | `kebab-case.vue` | `role-list.vue` |

### 禁止

```
MessageHelper.ts        # ❌ PascalCase 文件名
messageHelper.ts        # ❌ camelCase 文件名
roleApi.ts              # ❌ camelCase 文件名
```

## API 层规范

```ts
// role-api.ts
export const RoleApi = {
    list() {},
    create() {},
    update() {}
};

import { RoleApi } from "@/api/role-api";
```

## Store 规范

```ts
// user-store.ts
export const useUserStore = defineStore("user", {});
```

## 类型命名

- 类型名：`PascalCase`
- 接口名：`XxxDTO / XxxVO / XxxEntity`

## 总结

| 类型 | 规范 |
|---|---|
| 目录名 | `kebab-case` |
| 普通 TS 文件 | `kebab-case` |
| Vue 组件 | `PascalCase.vue` |
| composables | `use-xxx.ts` |
| API 文件 | `xxx-api.ts` |
| Store | `xxx-store.ts` |
