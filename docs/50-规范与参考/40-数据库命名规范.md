# 数据库命名规范

> 来源：[[00-项目总览|项目 VitePress 文档]]
>
> 适用范围：SPECTRA 平台所有数据库对象（表、字段、索引、约束）。

## 总体设计原则

### 1. 分层清晰

数据库对象必须体现系统层级与业务边界：

- **平台/核心能力层**：`SYS_`
- **基础资源域**：按资源域前缀（如 `ORG_`、`FILE_`）
- **业务域**：按业务域前缀（如 `PRJ_`、`ORD_`）

> 禁止所有业务表统一使用 `BIZ_`

### 2. 前缀即边界

一个前缀 = 一个逻辑域/子系统，用于权限、拆库、服务治理。

### 3. 命名稳定性

表名、字段名一旦上线，**原则上不修改**。

## 表命名规范

### 核心平台表（`SYS_`）

身份、权限、组织、配置、审计相关。

```
SYS_USER
SYS_ROLE
SYS_REL_USER_ROLE
SYS_ROLE_DATA_SCOPE
SYS_DICT
SYS_CONFIG
SYS_AUDIT_LOG
```

**强制规则**：SYS 表不得外键依赖任何业务表。

### 基础资源域表

```
ORG_ORGANIZATION
ORG_ORG_MEMBER
FILE_OBJECT
FILE_BUCKET
```

### 业务域表

```
{域前缀}_ + 表名

PRJ_PROJECT
PRJ_PROJECT_MEMBER
ORD_ORDER
ORD_ORDER_ITEM
```

### 关联/中间表

```
{域前缀}_REL_{主体1}_{主体2}

SYS_REL_USER_ROLE
PRJ_REL_PROJECT_USER
```

## 字段命名

### 通用字段

```sql
ID           UUID
CREATED_BY   UUID
CREATED_AT   TIMESTAMP(6) WITH TIME ZONE NOT NULL
UPDATED_BY   UUID
UPDATED_AT   TIMESTAMP(6) WITH TIME ZONE NOT NULL
DELETED      TIMESTAMP(6) WITH TIME ZONE
VERSION      BIGINT DEFAULT 0
```

### 外键字段

```
{引用表名}_ID

USER_ID
ROLE_ID
PROJECT_ID
```

### 枚举/类型字段

```
*_TYPE     → SCOPE_TYPE, TARGET_TYPE
*_STATUS   → USER_STATUS, ORDER_STATUS
*_FLAG     → default_flag, enabled_flag
```

## 索引命名

| 类型 | 格式 |
|---|---|
| 主键 | `PK_{TABLE_NAME}` |
| 唯一索引 | `UK_{TABLE_NAME}_{COLUMN}` |
| 普通索引 | `IDX_{TABLE_NAME}_{COLUMN}` |

## 禁止事项

- SYS 表引用业务表
- 不带前缀的业务表
- 表名使用动词
- 字段名使用拼音
- 不同语义字段复用同一名称

## 推荐前缀规划

```
SYS_   平台核心
ORG_   组织/租户
PRJ_   项目
WF_    工作流
ORD_   订单
FIN_   财务
FILE_  文件
```
