# 接口加解密方案

> 状态：✅ 已实现（15/15 任务完成） — 2026-07-11
>
> 基于 RSA + AES-GCM 混合加密的前后端接口数据加解密方案。

## 加密协议

```
请求加密：AES-256-GCM 随机密钥加密请求体 → RSA-OAEP 公钥加密 AES 密钥
响应加密：AES-256-GCM 随机密钥加密响应体 → RSA-OAEP 公钥加密 AES 密钥 → RSA 私钥签名
```

## 加密数据格式

```json
{
  "data": "AES-GCM 加密后的 Base64 密文",
  "key": "RSA-OAEP 加密后的 AES 密钥 (Base64)",
  "iv": "AES-GCM 初始化向量 (Hex)",
  "nonce": "随机数 (Base64)",
  "timestamp": 1234567890,
  "signature": "RSA 签名 (Base64)"
}
```

## 响应处理流程

```
Controller 返回业务数据
  → ResponseEncryptAdvice (@Order HIGHEST_PRECEDENCE)
    → AES-GCM 加密数据 → RSA 加密 AES 密钥 → RSA 签名
    → 返回 {data, key, iv, nonce, timestamp, signature} Map
  → ResponseModifyAdvice (@Order HIGHEST_PRECEDENCE + 1)
    → 包装为 R.success(加密Map)
    → 最终: {code:200, msg:"success", data: {data:"...", key:"...", ...}}
```

## 请求处理流程

```
HTTP 请求到达
  → MessageConverter 准备反序列化
  → RequestDecryptAdvice.beforeBodyRead()
    → 检查是否为加密请求（含 data + key + iv 字段）
    → RSA 验签 → RSA 私钥解密 AES 密钥 → AES-GCM 解密 data
    → 返回解密后的 body 供后续反序列化
  → Controller 收到正常的 DTO
```

## 目录结构

```
spectra-admin/spectra-common/src/main/java/com/devops00/spectra/common/
├── utils/
│   ├── AESUtils.java          ← AES-256-GCM 加解密
│   ├── RSAUtils.java          ← RSA-2048-OAEP 加解密 + 签名/验签
│   └── SHA256Utils.java       ← SHA256 摘要 / HMAC / Nonce 生成
└── exception/
    └── EncryptException.java  ← 加密异常

spectra-admin/spectra-framework/src/main/java/com/devops00/spectra/framework/configure/mvc/
├── properties/
│   └── SMProperties.java               ← 配置 (spectra.system.sm.*)
├── MvcConfiguration.java               ← 注册 ResponseEncryptAdvice Bean
└── advice/
    ├── request/
    │   └── RequestDecryptAdvice.java    ← 请求体解密 Advice
    └── response/
        ├── ResponseEncryptAdvice.java   ← 响应体加密 Advice
        └── ResponseModifyAdvice.java    ← 统一 R 包装 Advice

spectra-ui/src/
├── utils/crypto/
│   ├── crypto-utils.ts       ← 一键加密/解密/签名/验签
│   ├── aes-utils.ts          ← AES-GCM 前端实现
│   └── ras-utils.ts          ← RSA 前端实现
└── plugin/request/
    └── http.ts               ← HTTP 客户端（集成 encryptRequest/decryptResponse）
```

## 配置

```yaml
# spectra-admin
spectra:
  system:
    sm:
      enabled: true
      public-key: "MIIBIjANBgkqhkiG9w0B..."
      private-key: "MIIEvQIBADANBgkqhkiG9w0B..."
```

```env
# spectra-ui .env
VITE_CRYPTO_ENABLED=true
VITE_RSA_PUBLIC_KEY=MIIBIjANBgkqhkiG9w0B...
VITE_RSA_PRIVATE_KEY=MIIEvQIBADANBgkqhkiG9w0B...
```
