# 数据库连接池配置

> 来源：[[00-项目总览|项目 VitePress 文档]]
>
> spectra-admin 支持 HikariCP 和 Druid 两种连接池，当前使用 HikariCP。

## HikariCP 配置（当前使用）

```yaml
spring:
  datasource:
    driver-class-name: org.postgresql.Driver
    url: ${DB_URL}
    username: ${DB_USERNAME}
    password: ${DB_PASSWORD}
    type: com.zaxxer.hikari.HikariDataSource
    hikari:
      data-source-properties:
        currentSchema: public
        stringtype: unspecified
        reWriteBatchedInserts: true
        binaryTransfer: true
        tcpKeepAlive: true
        connectTimeout: 10
        socketTimeout: 60
        ApplicationName: ${spring.application.name}
        loginTimeout: 10
        sslmode: disable
        prepareThreshold: 5
```

### 参数说明

| 参数 | 说明 |
|---|---|
| `currentSchema` | 当前 schema |
| `stringtype: unspecified` | 让 PostgreSQL 自行推断 String 参数类型 |
| `reWriteBatchedInserts: true` | 启用批量 INSERT 重写优化 |
| `binaryTransfer: true` | 使用二进制协议传输数据 |
| `tcpKeepAlive: true` | TCP 层 KeepAlive 探测 |
| `connectTimeout: 10` | 连接超时（秒） |
| `socketTimeout: 60` | Socket 读超时（秒） |
| `prepareThreshold: 5` | Server-side Prepared Statement 启用阈值 |

## Druid 配置（备用）

```yaml
spring:
  datasource:
    type: com.alibaba.druid.pool.DruidDataSource
    druid:
      initial-size: 5
      min-idle: 5
      max-active: 20
      max-wait: 60000
      validation-query: SELECT 1
      test-while-idle: true
      test-on-borrow: false
      test-on-return: false
      time-between-eviction-runs-millis: 60000
      min-evictable-idle-time-millis: 300000
      pool-prepared-statements: false
      filters: stat,wall,slf4j
      stat-view-servlet:
        enabled: true
        url-pattern: /druid/*
      web-stat-filter:
        enabled: true
        url-pattern: /*
        exclusions: "*.js,*.css,*.png,*.jpg,*.gif,*.ico,/druid/*"
```
