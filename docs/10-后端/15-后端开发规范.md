# 后端开发规范

> spectra-admin 后端 Java 代码的统一编写规范。所有后端代码编写必须遵循本文档。

## 1. 注释规范

- 使用三斜杠（`///`）注释，而非 Javadoc 块注释
- 每个 Java 文件必须包含 Apache License 2.0 头部
- 类注释必须包含 `@author`、`@version`、`@since`

```java
/// 表单定义Service实现
///
/// @author yangxj96
/// @version 1.0
/// @since 2026/7/17
```

## 2. 包结构

### 简单模块（扁平结构）

适用于实体少、业务简单的模块（如 ai、upload、workflow）：

```
com.devops00.spectra.{module}
├── {Module}Module.java              # 模块入口（@ComponentScan + @MapperScan）
├── configuration/                    # 配置类
├── controller/                       # REST 端点
├── javabean/
│   ├── converter/                    # MapStruct 转换器
│   ├── entity/                       # 数据库实体
│   ├── from/                         # 请求表单对象
│   └── vo/                           # 响应视图对象
├── mapper/                           # MyBatis-Plus Mapper
├── service/                          # 业务逻辑接口
│   └── impl/                         # Service 实现
└── properties/                       # 配置属性（可选）
```

### 复杂模块（子域拆分）

适用于实体多、业务复杂的模块（如 core、oa）：

```
com.devops00.spectra.{module}
├── {Module}Module.java
├── {subdomain}/                      # 子域（如 auth/、system/、user/）
│   ├── controller/
│   ├── javabean/
│   │   ├── converter/
│   │   ├── entity/
│   │   ├── from/
│   │   └── vo/
│   ├── mapper/
│   └── service/
│       └── impl/
├── common/                           # 跨子域共享代码（可选）
└── configuration/                    # 模块级配置
```

## 3. Controller 规范

### 基本规则

- 统一 `@RequiredArgsConstructor` + `private final` 构造器注入
- 瘦 Controller：只做请求转发，不包含任何业务逻辑
- 禁止返回 `Object`，必须返回具体类型（`IPage<XxxVO>`、`XxxVO`、`List<XxxVO>`、`void`）
- 统一方法命名：`created`/`modify`/`deleteById`/`page`

### 注解使用

| 注解 | 规则 |
|---|---|
| `@ULog` | **所有接口方法必须加**。GET 用简短描述，POST/PUT/DELETE 用详细描述 |
| `@PreAuthorize` | **所有接口必须加**。公开接口用 `permitAll()` 显式标注 |
| `@Validated` | **所有写操作必须加**。使用 `Verify.Insert.class` 或 `Verify.Update.class` 分组 |
| `@Slf4j` | **必须加**。使用 `log.debug()` 记录入参（较低级别） |
| `version` | Mapping 注解中统一 `version = "1.0.0+"` |

### @ULog 描述格式

统一中文动宾短语：

| HTTP 方法 | 描述风格 | 示例 |
|---|---|---|
| GET | 简短 | `'查询表单列表'`、`'获取流程定义图'`、`'查询待办任务'` |
| POST | 详细 | `'创建表单定义'`、`'部署流程定义'`、`'启动流程'` |
| PUT | 详细 | `'更新表单定义'`、`'修改角色'` |
| DELETE | 详细 | `'删除表单定义'`、`'删除用户'` |

### @PreAuthorize 规则

- 所有接口必须加
- 公开接口用 `@PreAuthorize("permitAll()")` 显式标注
- 权限控制用 `@PreAuthorize("hasPermission(null, 'MODULE:ACTION')")`
- 角色控制用 `@PreAuthorize("hasRole('ROLE_xxx')")`

### @Slf4j 规则

- Controller 层必须加 `@Slf4j`
- 使用较低级别 `log.debug()` 记录请求入参
- Controller 层不写业务日志（业务日志由 `@ULog` 和 Service 层负责）

### 典型模板

```java
@Slf4j
@RestController
@RequestMapping("/workflow/form-definitions")
@RequiredArgsConstructor
public class FormDefinitionController {

    private final FormDefinitionService formDefinitionService;

    @ULog("'查询表单列表'")
    @GetMapping(value = "", version = "1.0.0+")
    @PreAuthorize("isAuthenticated()")
    public IPage<FormDefinitionVO> page(PageFrom page, FormPageFrom params) {
        return formDefinitionService.page(page, params);
    }

    @ULog("'创建表单定义'")
    @PostMapping(value = "", version = "1.0.0+")
    @PreAuthorize("hasPermission(null, 'WF_FORM:INSERT')")
    public void created(@Validated(Verify.Insert.class) @RequestBody FormDefinitionSaveFrom from) {
        formDefinitionService.created(from);
    }
}
```

## 4. Service 规范

### 接口

- 接口继承 `BaseService<Entity>`（**禁止直接继承 MyBatis-Plus 的 `IService`**）
- 方法声明使用 `created`/`modify`/`deleteById`/`page` 等命名
- 返回类型使用具体 VO 类型

```java
public interface FormDefinitionService extends BaseService<FormDefinition> {
    IPage<FormDefinitionVO> page(PageFrom page, FormPageFrom params);
    void created(FormDefinitionSaveFrom from);
    void modify(UUID id, FormDefinitionSaveFrom from);
    void deleteById(UUID id);
}
```

### 实现类

- 继承 `BaseServiceImpl<Mapper, Entity>` 并实现对应接口
- 必须加 `@Slf4j`、`@Service`
- 详细日志：关键业务节点用 `log.info()`，异常用 `log.error()`
- 写操作必须加 `@Transactional`
- 异常消息统一中文
- 字段注入 Converter 负责 From→Entity 和 Entity→VO 转换

```java
@Slf4j
@Service
@RequiredArgsConstructor
public class FormDefinitionServiceImpl
        extends BaseServiceImpl<FormDefinitionMapper, FormDefinition>
        implements FormDefinitionService {

    private final FormConverter formConverter;

    @Override
    @Transactional
    public void created(FormDefinitionSaveFrom from) {
        var entity = formConverter.toEntity(from);
        if (!this.save(entity)) {
            throw new DataSaveException("创建表单定义失败");
        }
        log.info("创建表单定义成功: id={}, name={}", entity.getId(), entity.getName());
    }
}
```

## 5. 异常处理

### 异常类型

统一使用项目自定义异常，**禁止裸 RuntimeException**：

| 异常类 | 说明 | 使用场景 |
|---|---|---|
| `DataNotExistException` | 数据不存在 | 查询/更新/删除时实体不存在 |
| `DataSaveException` | 数据保存失败 | insert/update 操作失败 |
| `EntityUpdateException` | 实体更新失败 | updateById 返回 false |
| `BuiltinDataException` | 内置数据不可操作 | 删除/修改系统内置数据 |
| `DefaultDataException` | 默认数据不可操作 | 删除默认角色/用户等 |

### 异常消息

- 统一中文（用户友好，前端可直接展示给用户）
- 示例：`"表单定义不存在"`、`"创建表单定义失败"`、`"内置角色,不可删除"`

### 异常使用原则

```java
// 正确：使用自定义异常
var entity = this.getById(id);
if (entity == null) {
    throw new DataNotExistException("表单定义不存在");
}

// 错误：禁止裸 RuntimeException
if (entity == null) {
    throw new RuntimeException("表单定义不存在");
}
```

## 6. 实体规范

- UUID 主键：`@TableId(type = IdType.INPUT)`
- 审计字段：`createdBy`、`createdAt`、`updatedBy`、`updatedAt`（继承 `BaseEntity`）
- 软删除：`Instant deleted`（null = 未删除）（继承 `BaseEntity`）
- 乐观锁：`@Version`（继承 `BaseEntity`）
- 默认排序：`@OrderBy` 设置
- 表名注解：`@TableName("xxx")`
- 字段注解：`@TableField("xxx")`

```java
@Getter
@Setter
@ToString
@TableName("wf_form_definition")
public class FormDefinition extends BaseEntity {

    @TableField("name")
    private String name;

    @TableField("code")
    private String code;
}
```

## 7. From 对象规范

- 统一后缀 `From`（**非 Form**）
- 包路径：`javabean/from/`
- 必填字段必须加 `@NotBlank`/`@NotNull` 等校验注解
- 使用 `@Data` 注解
- 提供清晰的中文 `message` 参数

```java
@Data
public class FormDefinitionSaveFrom {

    @NotBlank(message = "表单名称不能为空")
    private String name;

    @NotBlank(message = "规则JSON不能为空")
    private String ruleJson;

    private String description;  // 可选字段不加校验注解
}
```

## 8. VO 对象规范

- 统一后缀 `VO`
- 包路径：`javabean/vo/`
- 使用 `@Data` 注解
- 分页查询返回 `IPage<XxxVO>`

## 9. Converter 规范（MapStruct）

### 必须使用

所有模块必须使用 MapStruct Converter 进行对象转换，禁止手动 setter。

### 定义方式

```java
@Mapper(uses = TimeMapper.class, config = GlobalMapperConfig.class)
public interface FormConverter {

    /// 实体转 VO
    FormDefinitionVO toVO(FormDefinition source);

    /// From 转实体
    FormDefinition toEntity(FormDefinitionSaveFrom source);

    /// 更新已有实体
    void updateEntity(FormDefinitionSaveFrom source, @MappingTarget FormDefinition target);
}
```

### 包路径

统一放在 `javabean/converter/`

### 全局配置

引用 `GlobalMapperConfig.class` 和 `TimeMapper.class`，统一由 `spectra-framework` 提供。

## 10. 方法命名规范

| 操作 | 方法名 | 说明 |
|---|---|---|
| 创建 | `created` | 新增数据 |
| 更新 | `modify` | 修改数据 |
| 删除 | `deleteById` | 根据 ID 删除 |
| 分页查询 | `page` | 返回 `IPage<VO>` |
| 批量保存 | `saveBatch` | 批量插入 |
| 批量更新 | `updateBatch` | 批量更新 |
| 批量删除 | `deleteBatch` | 批量删除 |
| Upsert | `upsert` | 按唯一键去重的插入/更新 |

## 11. 日志规范

### 日志级别使用

| 级别 | Controller 层 | Service 层 |
|---|---|---|
| `log.debug()` | 记录请求入参 | 查询条件、参数详情 |
| `log.info()` | — | 关键业务节点（创建成功、部署成功等） |
| `log.warn()` | — | 业务校验失败、降级处理 |
| `log.error()` | — | 异常捕获、系统错误 |

### 日志消息格式

- 使用中文
- 关键节点必须记录实体 ID
- 示例：`log.info("创建表单定义成功: id={}, name={}", entity.getId(), entity.getName())`

## 12. 公开接口处理

- Controller 上加 `@PreAuthorize("permitAll()")` 显式标注
- SecurityConfig 中配置白名单作为兜底
- 两者结合使用，确保安全性

## 13. 模块入口规范

每个业务模块必须有一个 `*Module.java` 入口类：

```java
@ComponentScan("com.devops00.spectra.workflow")
@MapperScan("com.devops00.spectra.workflow.**.mapper")
public class WorkflowModule {
}
```
