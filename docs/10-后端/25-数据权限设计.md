# 数据权限设计

> 二维数据权限过滤框架，基于 MyBatis-Plus DataPermissionInterceptor 自动注入 WHERE 条件。
>
> 模块：`spectra-common` + `spectra-framework` + `spectra-core`

## 数据范围类型

```java
public enum DataScopeType {
    GLOBAL(0, "全局"),              // 超管赋予，查看所有数据
    SELF(1, "本人"),                // 只能看自己创建的
    DEPT(2, "部门"),                // 看本部门数据
    DEPT_AND_CHILDREN(3, "部门及子部门"), // 本部门+所有子部门
    CUSTOM(4, "自定义");            // 手动指定可见部门
}
```

## 设计规则

| 规则 | 说明 |
|---|---|
| 用户覆盖角色 | `sys_user_data_scope` 有值时，忽略角色级别的 `sys_role_data_scope` |
| 多角色取最大 | 用户未自定义时，遍历所有角色 scope，取范围最大者（GLOBAL > DEPT_AND_CHILDREN > DEPT > CUSTOM > SELF） |
| GLOBAL 仅超管赋予 | GLOBAL 只能由超管分配给特定用户/角色，管理员默认无 GLOBAL |
| SYS_ 表不过滤 | SYS_ 前缀系统表不参与数据范围过滤 |
| 无 `@DataScope` 不拦截 | Entity 上无 `@DataScope` 注解的表不注入 WHERE |

## 二维过滤模型

```
结构维度（部门归属）                   关系维度（多对多关联）
─────────────────                   ─────────────────
SELECT * FROM oa_meeting            SELECT * FROM oa_meeting
WHERE (                              WHERE (
    department_id IN (dept_ids)         ...
    OR created_by = current_user        OR id IN (
)                                           SELECT meeting_id FROM oa_meeting_participant
    ┌─ DEPT → 本部门                        WHERE user_id = ?
    ├─ DEPT_AND_CHILDREN → 本+子部门      )
    ├─ SELF → created_by              )
    └─ CUSTOM → 指定目标
```

## `@DataScope` 注解

```java
@Documented
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
public @interface DataScope {

    /// 结构维度过滤字段名，默认 department_id
    String column() default "department_id";

    /// 是否忽略（SYS_ 表等）
    boolean ignore() default false;

    /// 关系维度——关联表
    Relation[] relations() default {};

    @interface Relation {
        String table();                    // 关联表名
        String joinColumn();              // 关联表的外键字段
        String userColumn() default "user_id"; // 关联表的用户ID字段
        String mainColumn() default "id"; // 主表用于 IN 匹配的列
    }
}
```

### 使用示例

```java
// 仅结构维度
@DataScope
public class Asset extends BaseEntity { }

// 仅 user_id（本人可见）
@DataScope(column = "user_id")
public class MeetingParticipant extends BaseEntity { }

// 结构 + 参会人关系
@DataScope(relations = {
    @DataScope.Relation(
        table = "oa_meeting_participant",
        joinColumn = "meeting_id",
        userColumn = "user_id"
    )
})
public class Meeting extends BaseEntity { }

// 跨表关联（会议纪要通过 meeting_id 而非 id 匹配）
@DataScope(column = "department_id", relations = {
    @DataScope.Relation(
        table = "oa_meeting_participant",
        joinColumn = "meeting_id",
        userColumn = "user_id",
        mainColumn = "meeting_id"
    )
})
public class MeetingRecord extends BaseEntity { }
```

## 核心流程

```
用户登录
  │
  └→ SecurityUserHelper.toSecurityUser()
       └→ DataScopeResolver.resolve(userId)
            ├─ 查 sys_user_data_scope（有 → 覆盖角色）
            └─ 查 sys_role_data_scope（无 → 多角色取最大）
                 └→ 存入 SecurityUser.dataScopeType / departmentId / targetIds

请求任意 SQL
  │
  └→ DataScopeInnerInterceptor.getSqlSegment()
       ├─ GLOBAL → return null（不过滤）
       ├─ 无 @DataScope → return null
       ├─ ignore = true → return null
       └─ 生成 WHERE：
            ├─ 结构维度：column IN (targetIds) 或 column = userId
            ├─ 关系维度：mainColumn IN (SELECT joinColumn FROM relationTable WHERE userColumn = ?)
            └─ 两者 OR 连接
```

## 关键文件路径

| 文件 | 路径 | 说明 |
|---|---|---|
| `@DataScope` 注解 | `spectra-common/.../annotation/DataScope.java` | 数据范围注解定义 |
| `DataScopeProvider` | `spectra-common/.../mybatis/DataScopeProvider.java` | scope 提供者接口 |
| `DataScopeResolver` | `spectra-core/.../impl/DataScopeResolver.java` | scope 解析实现（合并规则） |
| `DataScopeInnerInterceptor` | `spectra-framework/.../interceptor/DataScopeInnerInterceptor.java` | SQL 拦截器（WHERE 生成） |
| `SecurityUser` | `spectra-security-base/.../entity/SecurityUser.java` | 携带 scope 的认证实体 |
| `SecurityUserHelper` | `spectra-core/.../impl/SecurityUserHelper.java` | 登录时加载 scope |
| `MyBatisPlusConfiguration` | `spectra-framework/.../MyBatisPlusConfiguration.java` | 注册拦截器 |
| OA 建表 DDL | `spectra-admin/document/db/oa/01-create-tables.sql` | 含 department_id |
