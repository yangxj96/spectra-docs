# 模块清单

> 所有子项目及其模块的一句话职责描述。

## 后端模块 (spectra-admin)

### 基础层

| 模块 | 路径 | 职责 |
|---|---|---|
| spectra-config | `spectra-config/` | 统一配置文件，所有 application-*.yml 集中管理 |
| spectra-common | `spectra-common/` | 公共基础：BaseEntity、DTO、工具类、MyBatis-Plus 配置 |
| spectra-framework | `spectra-framework/` | 框架层：MVC、JSON、Redis、缓存、安全配置 |

### 启动与自动配置

| 模块 | 路径 | 职责 |
|---|---|---|
| spectra-launch | `spectra-launch/` | Spring Boot 启动入口，业务入口 |
| spectra-starter | `spectra-starter/` | 自动配置 Starter（Security/Log/AI） |

### 业务模块

| 模块 | 路径 | 职责 |
|---|---|---|
| spectra-core | `modules/spectra-core/` | 核心能力：用户/角色/权限/部门/菜单/字典/区域/日志/配置 |
| spectra-oa | `modules/spectra-oa/` | OA 办公：资产/考勤/日历/通讯录/合同/文档/会议/公告/报表 |
| spectra-upload | `modules/spectra-upload/` | 文件上传：本地存储 + S3，分片上传 |
| spectra-workflow | `modules/spectra-workflow/` | 工作流：Flowable 流程引擎，审批/任务/历史 |
| spectra-ai | `modules/spectra-ai/` | AI 集成：LangChain4j + RAG 检索增强生成 |

## 模块依赖关系

```
spectra-oa ──→ spectra-core
              spectra-workflow ──→ spectra-core
              spectra-upload ──→ spectra-common
              spectra-ai ──→ spectra-common

spectra-launch ──→ spectra-modules
                 spectra-starter ──→ spectra-framework
                                   spectra-modules ──→ spectra-framework
                                                     spectra-common
```
