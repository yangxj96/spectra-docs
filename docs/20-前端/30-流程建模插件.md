# 流程建模插件 — logicflow-flowable

> 基于 [LogicFlow](https://logicflow.org) 的 BPMN 2.0 流程建模插件，无缝对接 [Flowable](https://www.flowable.org) 工作流引擎。路径：`logicflow-flowable/`。

## 概述

| 属性 | 值 |
|---|---|
| 仓库 | https://github.com/yangxj96/logicflow-flowable |
| npm 包名 | `@yangxj96/logicflow-flowable` |
| 版本 | 0.0.4-dev |
| 许可 | Apache 2.0 |
| 用途 | spectra-ui 工作流流程设计、BPMN XML 导入/导出 |

## 技术栈

| 技术 | 版本 | 用途 |
|---|---|---|
| LogicFlow | ^2.2.0 | 流程图引擎核心 |

| TypeScript | ^5.9.3 | 类型安全 |
| tsup | ^8.5.1 | 构建打包（ESM + CJS） |

| Prettier | ^3.7.4 | 代码格式化 |

## 核心功能

### BPMN 元素支持矩阵

#### 事件（Events）

| BPMN 元素 | 类型 | 状态 | 说明 |
|---|---|---|---|
| 开始事件 | `bpmn:startEvent` | ✅ | 流程起点，细线圆 |
| 结束事件 | `bpmn:endEvent` | ✅ | 流程终点，粗线圆 |
| 中间捕获事件 | `bpmn:intermediateCatchEvent` | ✅ | 双圆环 |
| 中间抛出事件 | `bpmn:intermediateThrowEvent` | ✅ | 双圆环，内圈填充 |
| 边界事件 | `bpmn:boundaryEvent` | ✅ | 虚线圆环 |

#### 任务（Tasks）

| BPMN 元素 | 类型 | 状态 | 说明 |
|---|---|---|---|
| 用户任务 | `bpmn:userTask` | ✅ | 人形图标 |
| 服务任务 | `bpmn:serviceTask` | ✅ | 齿轮图标 |
| 脚本任务 | `bpmn:scriptTask` | ✅ | 代码图标 |
| 接收任务 | `bpmn:receiveTask` | ✅ | 信封图标 |
| 手工任务 | `bpmn:manualTask` | ✅ | 手形图标 |
| 业务规则任务 | `bpmn:businessRuleTask` | ✅ | 表格图标 |

#### 网关（Gateways）

| BPMN 元素 | 类型 | 状态 | 说明 |
|---|---|---|---|
| 排他网关 | `bpmn:exclusiveGateway` | ✅ | 菱形 + X |
| 包容网关 | `bpmn:inclusiveGateway` | ✅ | 菱形 + ○ |
| 并行网关 | `bpmn:parallelGateway` | ✅ | 菱形 + + |

#### 子流程与调用

| BPMN 元素 | 类型 | 状态 | 说明 |
|---|---|---|---|
| 嵌入式子流程 | `bpmn:subProcess` | ✅ | 容器节点，可缩放 |
| 展开子流程 | `bpmn:expandedSubProcess` | ✅ | 虚线边框 |
| 调用活动 | `bpmn:callActivity` | ✅ | 粗边框 + 波浪线 |

#### 连线（Flows）

| BPMN 元素 | 类型 | 状态 | 说明 |
|---|---|---|---|
| 序列流 | `bpmn:sequenceFlow` | ✅ | 默认连线类型 |

### BPMN XML 导入/导出

#### 导出 (`toBpmnXml`)

将 LogicFlow 画布导出为 Flowable 兼容的 BPMN 2.0 XML：

```typescript
import Flowable from '@yangxj96/logicflow-flowable';

// 导出 BPMN XML
const xml = Flowable.toBpmnXml(lf);
console.log(xml);
// <?xml version="1.0" encoding="UTF-8"?>
// <definitions xmlns="http://www.omg.org/spec/BPMN/20100524/MODEL" ...>
//   <process id="process_1" name="我的流程" isExecutable="true">
//     <startEvent id="node_xxx" name="开始" />
//     <userTask id="node_yyy" name="审批" />
//     <sequenceFlow id="edge_zzz" sourceRef="node_xxx" targetRef="node_yyy" />
//   </process>
// </definitions>
```

**导出特性**：
- 支持所有 BPMN 元素属性导出
- 支持 Flowable 扩展属性（`flowable:assignee` 等）
- 自动生成 BPMN DI 位置信息
- 支持 CDATA 包裹的复杂内容

#### 导入 (`fromBpmnXml`)

从 BPMN XML 导入流程图到 LogicFlow 画布：

```typescript
import Flowable from '@yangxj96/logicflow-flowable';

// 导入 BPMN XML
const result = Flowable.fromBpmnXml(xmlString, lf);
if (result.success) {
    console.log(`成功导入 ${result.nodes.length} 个节点`);
} else {
    console.error(result.message);
}
```

**导入特性**：
- 支持标准 BPMN 2.0 XML
- 支持 Flowable 扩展属性
- 自动解析 BPMN DI 位置信息
- 无位置信息时自动网格布局
- 自动处理重复 ID
- XML 属性去重修复

## 插件初始化流程

```
FlowablePlugin 构造函数
    │
    ├─ 1. initContext()         → 初始化流程元数据（id, name, isExecutable）
    │
    ├─ 2. initElements()        → 注册所有 BPMN 元素
    │   ├─ registerEventNodes()      → 事件节点（5种）
    │   ├─ registerTaskNodes()       → 任务节点（4种）
    │   ├─ registerSequenceEdges()   → 序列流连线
    │   ├─ registerGatewayNodes()    → 网关节点（3种）
    │   └─ registerSubProcessNodes() → 子流程节点（3种）
    │
    ├─ 3. initPlugins()         → 注册插件能力
    │   ├─ registerContextMenu()     → 右键菜单
    │   └─ DynamicGroup              → 子流程容器支持
    │
    ├─ 4. initDefaultConfig()   → 默认配置
    │   ├─ 默认连线类型 = bpmn:sequenceFlow
    │   └─ 启用节点缩放（resize）
    │
    └─ 5. bindEvents()          → 事件绑定
        ├─ graph:rendered → initPanels() → 挂载 DND 和属性面板
        └─ connection:not-allowed → 控制台警告
```

## 开发工作流

### 在 spectra-ui 中调试

1. **启动监听模式**（logicflow-flowable/ 目录）：
   ```bash
   pnpm run dev
   ```
   这会监听 `src/` 变化并自动重新构建。

2. **启动 spectra-ui 开发服务器**（spectra-ui/ 目录）：
   ```bash
   pnpm start
   ```

3. **修改 logicflow-flowable 源码**后，构建产物会自动更新到 spectra-ui。

### 添加新节点类型

1. **创建节点文件**（以 UserTask 为例）：
   ```
   src/elements/nodes/tasks/user-task/
   ├── model.ts      # 数据模型，继承 TaskBaseModel
   ├── view.ts       # 视图，继承 TaskBaseView
   └── index.ts      # 导出 registerUserTask 函数
   ```

2. **在 `core/constants.ts` 添加类型**：
   ```typescript
   USER_TASK: bpmn("userTask"),
   ```

3. **在 `elements/nodes/tasks/index.ts` 注册**：
   ```typescript
   export function registerTaskNodes(lf: LogicFlow) {
       registerUserTask(lf);
       // ...
   }
   ```

4. **在 `features/schema/` 添加属性 schema**：
   定义节点可编辑的属性字段。

### 添加新属性面板

1. 在 `panel/property/renderer/` 添加渲染器
2. 在 `features/schema/` 添加对应的 schema 定义
3. 在 `features/export/builder.ts` 处理导出逻辑

## 配置选项

```typescript
interface FlowablePluginOptions {
    panel?: {
        dnd?: HTMLElement;      // 拖拽面板容器
        property?: HTMLElement; // 属性面板容器
    };
}
```

**使用示例**：
```vue
<script setup>
import { ref, onMounted } from 'vue';
import LogicFlow from '@logicflow/core';
import Flowable from '@yangxj96/logicflow-flowable';

const dndRef = ref(null);
const propertyRef = ref(null);
const lfRef = ref(null);

onMounted(() => {
    const lf = new LogicFlow({
        container: document.getElementById('graph-container'),
    });

    lf.use(Flowable.Plugin, {
        panel: {
            dnd: dndRef.value,
            property: propertyRef.value
        }
    });

    lfRef.value = lf;
});
</script>

<template>
    <div ref="dndRef" class="dnd-panel" />
    <div id="graph-container" />
    <div ref="propertyRef" class="property-panel" />
</template>
```

## 与 Flowable 后端集成

### 典型工作流

1. **前端设计**：用户在 spectra-ui 中使用 logicflow-flowable 设计流程
2. **导出 XML**：调用 `toBpmnXml()` 生成 BPMN 2.0 XML
3. **后端部署**：将 XML 发送到 spectra-admin，由 Flowable 引擎部署执行
4. **流程运行**：Flowable 引擎执行流程实例，处理任务节点
5. **任务查询**：前端查询待办任务，显示流程实例状态

### 数据流

```
spectra-ui                          spectra-admin
    │                                     │
    │  1. 设计流程                         │
    │  ┌─────────────┐                    │
    │  │ LogicFlow    │                    │
    │  │ + Flowable   │                    │
    │  │   插件       │                    │
    │  └─────────────┘                    │
    │         │                           │
    │  2. toBpmnXml()                     │
    │         │                           │
    │  3. POST /api/workflow/deploy ──────>│
    │                                     │  4. Flowable 部署
    │                                     │     流程定义
    │  5. GET /api/workflow/tasks <───────│
    │         │                           │
    │  6. 显示任务列表                     │
```

## 节点行为规则

> 所有规则均由 `NodeBehavior` 定义（`src/features/behaviors/`），通过 `BehaviorsEngine` 执行校验。

### 校验阶段

| 阶段 | 触发时机 | 目的 |
|------|----------|------|
| 创建连线 | `edge:add` | 阻止绝对非法的连线产生，只校验拓扑与类型合法性 |
| 全流程校验 | 发布/保存流程 | 保证流程可执行，校验条件完整性、默认连线、最小/最大数量 |

### 通用规则

| 规则项 | 说明 |
|--------|------|
| `allowIn` / `allowOut` | 是否允许作为入线/出线节点 |
| `minIn` / `maxIn` | 最小/最大入线数量 |
| `minOut` / `maxOut` | 最小/最大出线数量 |
| `allowSourceTypes` / `allowTargetTypes` | 允许的来源/目标节点类型 |

### 事件节点规则

| 节点 | 入线 | 出线 | 特殊约束 |
|------|------|------|----------|
| 开始事件 | ❌ | ✅ 1条 | 必须且只能有一条出线 |
| 结束事件 | ✅ ≥1 | ❌ | 不允许出线 |
| 中间捕获事件 | ✅ | ✅ | 必须有入线和出线 |
| 中间抛出事件 | ✅ | ✅ | 必须有入线和出线 |
| 边界事件 | ❌ | ✅ | 依附于活动节点，不允许入线 |

### 任务节点规则

| 节点 | 入线 | 出线 | 特殊约束 |
|------|------|------|----------|
| 用户任务 | ≥1 | 1 | 只能有一条出线 |
| 服务任务 | ≥1 | 1 | 出线数量为 1 |
| 脚本任务 | ≥1 | 1 | 出线数量为 1 |
| 接收任务 | ≥1 | ≤1 | 允许无出线（流程可暂停） |

### 子流程与调用规则

| 节点 | 入线 | 出线 | 特殊约束 |
|------|------|------|----------|
| 嵌入式子流程 | ≥1 | ≥1 | 可以有多条出线（并行分支） |
| 调用活动 | ≥1 | 1 | 出线数量为 1 |

> 注：BPMN 2.0 标准中，展开/折叠是视图行为，不作为独立类型。

### 网关节点规则

| 节点 | 入线 | 出线 | 特殊约束 |
|------|------|------|----------|
| 排他网关 | ≥1 | ≥2 | 至少 2 条出线，每条非默认出线必须配置条件，必须且只能存在一条默认出线 |
| 包容网关 | ≥1 | ≥2 | 至少 2 条出线，所有出线必须配置条件，不允许默认连线 |
| 并行网关 | ≥1 | ≥2 | 至少 2 条出线，不允许配置条件，不允许默认连线 |

### 边（Edge）规则

**创建阶段**：
- 禁止重复连线：同一 source → target 仅允许一条
- 类型匹配：source / target 类型必须符合节点定义

**全流程阶段**：
- 条件完整性：排他 / 包容网关的条件校验
- 默认连线：排他网关默认线校验
- 并行约束：并行网关不允许条件

### 设计原则

- **连线创建 ≠ 流程完成**
- **创建阶段只做结构合法性校验**
- **发布阶段才做完整性校验**
- 所有行为规则集中在 `NodeBehavior`
- 所有执行逻辑集中在 `BehaviorsEngine`
