# 后端开发规范

> spectra-admin 后端开发的核心规范，确保代码风格和架构一致性。

## 分层职责

### Controller 层

**职责**：只处理 HTTP 请求转发，不包含业务逻辑。

- 接收请求参数（`@RequestBody`、`@RequestParam`、`@PathVariable`）
- 调用 Service 层方法
- 返回响应结果（VO 对象）

**禁止**：
- 在 Controller 中编写业务逻辑
- 直接操作数据库（Mapper）
- 使用 `Map` 作为响应返回值

### Service 层

**职责**：处理所有业务逻辑。

- 实现业务规则
- 事务管理（`@Transactional`）
- 数据校验
- 调用 Mapper 层

### Mapper 层

**职责**：数据访问。

- 继承 `BaseMapper<T>` 获得 CRUD 能力
- 自定义查询使用 `@Select` 注解或 XML 映射

## Controller 响应规范

### 使用 VO 对象

```java
// ✅ 正确：使用 VO 对象
@GetMapping("/config")
public CryptoConfigVO getConfig() {
    return new CryptoConfigVO(enabled, serverPublicKey);
}

// ❌ 错误：使用 Map
@GetMapping("/config")
public Map<String, Object> getConfig() {
    Map<String, Object> result = new HashMap<>();
    result.put("enabled", enabled);
    return result;
}
```

### VO 命名约定

- 使用 PascalCase + `VO` 后缀
- 示例：`CryptoConfigVO`、`UserDetailVO`、`ConfiguredVO`

### VO 结构

```java
@Data
@AllArgsConstructor
@NoArgsConstructor
public class CryptoConfigVO implements Serializable {
    @Serial
    private static final long serialVersionUID = 1L;

    private Boolean enabled;
    private String serverPublicKey;
}
```

## 方法命名规范

### 数据库 CRUD

| 操作 | 方法名 | 说明 |
|---|---|---|
| 新增 | `save` | 保存单条记录 |
| 更新 | `update` | 更新单条记录 |
| 删除 | `delete` | 删除单条记录 |
| 按 ID 查询 | `getById` | 根据主键查询 |
| 列表查询 | `list` | 查询列表 |

### 数据库 Upsert

| 操作 | 方法名 | 说明 |
|---|---|---|
| 插入/更新 | `upsert` | 按唯一键去重，存在则更新，不存在则插入 |

```java
// Service 接口
void upsert(String key, String value, String remarks);

// Service 实现
@Override
@Transactional
public void upsert(String key, String value, String remarks) {
    var existing = this.getOne(
            new LambdaQueryWrapper<Entity>().eq(Entity::getKey, key));
    if (existing != null) {
        existing.setValue(value);
        this.updateById(existing);
    } else {
        var entity = new Entity();
        entity.setKey(key);
        entity.setValue(value);
        this.save(entity);
    }
}
```

### 分页查询

| 操作 | 方法名 | 说明 |
|---|---|---|
| 分页 | `page` | 分页查询，返回 `IPage<VO>` |

### 批量操作

| 操作 | 方法名 | 说明 |
|---|---|---|
| 批量新增 | `saveBatch` | 批量保存记录 |
| 批量更新 | `updateBatch` | 批量更新记录 |
| 批量删除 | `deleteBatch` | 批量删除记录 |
