# 工作流模块

> Flowable 流程引擎集成：审批流程、任务管理、历史记录。

## 模块简介

`spectra-workflow` 是 Spectra 后端平台的工作流模块，提供统一的流程引擎能力。

## 目录结构

```
spectra-workflow/
├── controller/     ← REST 端点
├── service/        ← 业务逻辑
│   └── impl/
├── mapper/         ← MyBatis-Plus Mapper
├── configuration/  ← 工作流配置
└── javabean/
    ├── entity/     ← 数据库实体
    ├── from/       ← 请求表单对象
    └── vo/         ← 响应视图对象
```

## 核心能力

### 流程定义

- 流程模型管理
- 流程版本管理
- 流程发布与部署

### 流程实例

- 流程实例启动
- 流程状态跟踪
- 流程执行记录

### 任务管理

- 待办任务、已办任务
- 任务指派、任务处理
- 签收、完成、转办

### 审批记录

- 审批记录、审批意见
- 审批时间、审批结果
- 流程审计、审批追溯

### 表单数据

- 动态表单支持
- 业务数据绑定

## 业务使用方式

```
1. 创建业务数据（如请假申请）
2. 启动流程实例 → workflow.startProcess()
3. 用户审批任务 → workflow.completeTask()
4. 流程结束 → 更新业务状态（APPROVED/REJECTED）
```

## 技术实现

- 基于 Flowable 流程引擎
- 支持 BPMN 流程定义
- 提供统一接口供业务模块调用
- 与 LogicFlow 前端流程设计器集成

## 模块关系

```
spectra-workflow ← 被以下模块依赖
└── spectra-oa（审批流程）
```

## API 端点

| Controller | 路径 | 说明 |
|---|---|---|
| FormDefinitionController | `/workflow/form-definitions/**` | 表单定义管理（CRUD + 版本管理） |
| ProcessDefinitionController | `/workflow/process-definitions/**` | 流程定义查询/挂起/激活/获取资源/部署 |
| ProcessInstanceController | `/workflow/process-instances/**` | 流程实例启动/查询/终止/流程图 |
| TaskController | `/workflow/tasks/**` | 待办/已办/签收/完成/转办/委派/驳回 |
| RuntimeController | `/workflow/runtime/**` | 运行时状态查询（待实现） |
| HistoryController | `/workflow/history/**` | 历史记录查询（待实现） |
